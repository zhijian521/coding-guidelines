# 十二、工程化配置

工具配置的核心原则：**交给机器做的事，不要让人来纠结**。缩进、分号、引号这些格式问题不应该出现在 Code Review 讨论中——Prettier 和 ESLint 应该在你保存文件时就自动处理好了。

## 12.1 Prettier

```json
{
  "semi": true,
  "singleQuote": true,
  "trailingComma": "es5",
  "tabWidth": 2,
  "useTabs": false,
  "printWidth": 100,
  "bracketSpacing": true,
  "arrowParens": "always",
  "endOfLine": "lf"
}
```

## 12.2 Git Hooks

```json
{
  "lint-staged": {
    "*.{js,ts,jsx,tsx,vue}": [
      "eslint --fix",
      "prettier --write"
    ],
    "*.{css,scss}": [
      "stylelint --fix",
      "prettier --write"
    ]
  }
}
```

## 12.3 环境变量

| 文件 | 用途 |
|---|---|
| `.env` | 所有环境共享 |
| `.env.development` | 开发环境 |
| `.env.test` | 测试环境 |
| `.env.production` | 生产环境 |
| `.env.local` | 本地覆盖（不提交 Git） |

> 敏感信息（API Key、密钥）禁止出现在前端环境变量或代码中。所有敏感操作在服务端完成。

## 12.4 工程化补充

| 项 | 要求 |
|---|---|
| 包管理器 | 统一使用 pnpm；lock 文件（`pnpm-lock.yaml`）必须提交到 Git |
| browserslist | 在 `package.json` 中明确目标浏览器范围，避免生成不必要的 polyfill |
| .editorconfig | 项目根目录放置 `.editorconfig`，统一缩进、换行符、字符编码 |
| 路径别名 | 配置 `@/` 指向 `src/`，避免深层相对路径 `../../../` |
| npm scripts | `dev` `build` `lint` `type-check` 作为标准命令名，不自定义 |
