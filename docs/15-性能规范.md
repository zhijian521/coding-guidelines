# 十五、性能规范

性能优化不用过度追求——首屏加载在 2 秒以内、交互响应在 100ms 以内，对大部分项目就够了。关键是养成好习惯：不引入 500KB 的库只为了用一个函数，不把整个页面打包到一个 JS 文件里。

## 15.1 代码优化

```typescript
// 路由懒加载
const routes = [{
  path: '/dashboard',
  component: () => import('@/pages/Dashboard.vue'),
}];

// 按需引入
import { debounce } from 'lodash-es';  // 而非 import _ from 'lodash'

// 图片懒加载
<img src="image.jpg" loading="lazy" />

// 使用 WebP
<picture>
  <source srcset="img.webp" type="image/webp">
  <img src="img.jpg" />
</picture>
```

## 15.2 优化清单

| 方向 | 措施 |
|---|---|
| 代码分割 | 路由级懒加载、组件动态导入 |
| 图片优化 | WebP/AVIF、懒加载、`srcset` |
| 缓存 | API 数据缓存、静态资源强缓存 |
| 减少重渲染 | `useMemo` / `computed`、`useCallback` |
| 打包优化 | Tree Shaking、按需引入 |
| 长列表 | 虚拟滚动 |

## 15.3 性能指标与监控

以下为 Google Core Web Vitals 推荐阈值，新页面应在 CI 中自动检查：

| 指标 | 目标 | 说明 |
|---|---|---|
| LCP | ≤ 2.5s | 最大内容绘制：页面主要内容加载完成的时间 |
| INP | ≤ 200ms | 交互到下次绘制：用户操作到页面响应的延迟 |
| CLS | ≤ 0.1 | 累积布局偏移：页面加载过程中视觉稳定性的评分 |

## 15.4 资源加载策略

| 资源 | 策略 |
|---|---|
| 关键 CSS | 内联到 `<head>`，非关键 CSS 延迟加载 |
| 关键 JS | 异步加载（`async`），不阻塞渲染 |
| Hero 图片 | `fetchpriority="high"` 提前加载 |
| 字体文件 | `font-display: swap` + `preload` + 子集化 |
| 第三方脚本 | 异步加载、按需加载、设置性能预算（超过 50KB 需审批） |

## 15.5 Bundle 分析

每次大版本发布前运行 bundle 分析工具，检查是否有意外引入的大依赖包。

```shell
# Vite 项目
npx vite-bundle-visualizer

# Webpack 项目
npm install --save-dev webpack-bundle-analyzer
npx webpack-bundle-analyzer dist/stats.json

# 在线工具 — 引入新依赖前检查体积
# bundlephobia.com
```
