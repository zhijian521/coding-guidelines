# 十四、安全规范

前端安全问题一旦出事故就是大事——XSS 可以窃取用户 Token，敏感信息泄露可能导致服务器被入侵。这些规则不是可选项，违反任何一条 Code Review 直接打回。

## 14.1 XSS 防护

**危险做法：**
```javascript
// 直接插入 HTML
el.innerHTML = userInput;

// Vue: v-html
<div v-html="userInput"/>

// React: dangerouslySetInnerHTML
```

**安全做法：**
```javascript
// 安全插入文本
el.textContent = userInput;

// 使用 DOMPurify
import DOMPurify from 'dompurify';
const clean = DOMPurify.sanitize(input);
el.innerHTML = clean;
```

## 14.2 安全清单

| 项 | 要求 |
|---|---|
| XSS | 避免 `innerHTML` / `v-html`，必要时用 DOMPurify |
| HTTPS | 生产环境强制 HTTPS，所有 API 用 HTTPS |
| 敏感信息 | 禁止在前端硬编码密钥、密码 |
| URL 参数 | 传递时用 `encodeURIComponent` 编码 |
| CSP | 配置 Content Security Policy 头 |
| Token 存储 | Token 存 HttpOnly Cookie，禁止存 localStorage（防 XSS 窃取） |
| iframe 安全 | 嵌入外部页面必须加 `sandbox` 属性，限制权限 |
| 依赖扫描 | `npm audit` 或 Snyk 集成到 CI，高危漏洞阻断发布 |
| 输入校验 | 前端做格式校验提升体验，后端做安全校验拦截攻击——两者缺一不可 |
